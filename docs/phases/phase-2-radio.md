# Phase 2 — Personal Radio Station

## Goal
Transform MusicPhiles from a player into a personal radio station. The user's music becomes a broadcast, with a voice, identity, and programming.

---

## Core features (to be detailed)

- Voiced station identity — a personality reads station IDs and transitions
- News bulletins — scheduled or triggered news breaks
- Station bumps — short audio idents between tracks
- Programming logic — more than shuffle, less than an algorithm

---

## Dependencies
Requires Phase 1 to be complete and stable.

---

## Open questions
- Voice synthesis approach (TTS service, pre-recorded, user's own voice?)
- News source integration
- How much programming control does the user have?
- Does this require a lightweight backend, or can it stay frontend-only?

---

## What this phase explicitly excludes
- HelloTape / mixtape sharing (Phase 3)
- Multi-user or social features
- Public broadcasting
